# სერვის მენეჯერების რეპორტი (პროექტი × სართული)

**პროექტების მიხედვით სართულებზე მომაგრებული სერვის მენეჯერები** — ლისტი 82 (Bitrix24) იტვირთება ბრაუზერში, ტერმინალის გაშვების გარეშე.

## როგორ გამოვიყენოთ (დეპლოიზის შემდეგ)

1. საიტი უნდა იყოს დეპლოირებული (მაგ. [Vercel](https://vercel.com) – GitHub-თან დაკავშირებით ანაწილებს ავტომატურად).
2. გახსენით საიტის მისამართი ბრაუზერში (მაგ. `https://archi-managers.vercel.app`).
3. დააწკაპუნეთ **„რეპორტის ჩატვირთვა“** — მონაცემები ჩაიტვირთება და ცხრილი იმავე გვერდზე გამოჩნდება. ტერმინალში არაფრის გაშვება არ სჭირდება.

## რატომ 404-ს ვხედავდით

- ცალკე ფაილი `report.html` საიტზე არ იტვირთებოდა (არ იყო რეპოზიტორიაში).
- ახლა რეპორტი იგივე გვერდზეა (**index.html**): ღილაკით იტვირთება მონაცემები და ცხრილი იქვე ჩანს. 404 აღარ უნდა გამოჩნდეს.

## დეპლოი Vercel-ზე

1. რეპოზიტორია დაუკავშირდით [Vercel](https://vercel.com)-ს (Import from GitHub).
2. Deploy-ის შემდეგ გახსენით მიღებული ლინკი და დააწკაპუნეთ „რეპორტის ჩატვირთვა“.

(სურვილისამებრ Vercel-ის Project Settings → Environment Variables-ში დაამატეთ `BITRIX_WEBHOOK_URL` = `https://crm.archi.ge/rest/1/...` — თორემ კოდში ჩაწერილი ბრუნდება.)

## ფაილების მიმოხილვა

| ფაილი | დანიშნულება |
|-------|---------------|
| **index.html** | ერთი გვერდი: ღილაკი „რეპორტის ჩატვირთვა“ + ცხრილი. ტერმინალი არ სჭირდება. |
| **api/report.js** | Vercel serverless: Bitrix24-ის API-ის proxy (CORS/404-ის თავიდან ასაცილებლად). |
| **fetch-report.js** + **config.js** | ტერმინალიდან გაშვება (სურვილისამებრ), რეპორტი იწერება `report.html`-ში. |

## ველების მაპინგი

თუ ცხრილი ცარიელი ან არასწორია, Bitrix24-ში ველების ID-ები შეიძლება განსხვავებული იყოს. `index.html`-ში script-ის დასაწყისში არის `CONFIG.fields` — იქ შეგიძლიათ შეცვალოთ (მაგ. `PROPERTY_1472` → თქვენი პროექტის ველის ID). იგივე მაპინგია `config.js`-ში `fetch-report.js`-ისთვის.
